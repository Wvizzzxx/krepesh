# Система управления иконками и изображениями

## Описание
Единая система управления иконками (Font Awesome) и фото-заглушками (загруженные изображения), которые сохраняются в базе данных и доступны для использования по всей системе.

## Где найти

### Админка
- **URL**: http://localhost:3002/assets
- **Меню**: Боковое меню → **Иконки и изображения** (🎨 иконка)

## Функциональность

### 1. Просмотр
- Все элементы отображаются в виде сетки карточек
- Иконки (Font Awesome) отображаются на тёмном фоне с оранжевым цветом
- Заглушки (изображения) отображают загруженное фото
- Битые изображения заменяются SVG-заглушкой с названием
- Каждая карточка показывает:
  - Превью иконки/изображения
  - Название
  - Тип (Иконка или Заглушка) с цветным бейджем
  - Категория (если указана)
  - Описание
  - Код для использования (для иконок): `<Icon name="..." />`

### 2. Поиск и фильтрация
- **Поиск** по названию, описанию и категории
- Три вкладки для фильтрации:
  - **Все** — показывает все элементы
  - **🎯 Иконки** — только Font Awesome иконки
  - **🖼️ Заглушки** — только загруженные изображения

### 3. Добавление нового элемента
1. Нажмите кнопку **+ Добавить**
2. Выберите тип:
   - **Иконка (Font Awesome)** — выберите название иконки из выпадающего списка с предпросмотром
   - **Заглушка (изображение)** — введите название и загрузите файл изображения
3. Укажите категорию (опционально): header, footer, home, product, payment, delivery, other
4. Добавьте описание (опционально)
5. Для заглушек: укажите ширину и высоту (опционально)
6. Нажмите **Сохранить**

### 4. Редактирование
1. Нажмите на иконку редактирования (✏️) на карточке
2. Обновите нужные поля
3. Опционально загрузите новое изображение (для заглушек)
4. Нажмите **Сохранить**

### 5. Удаление
1. Нажмите на иконку удаления (🗑️) на карточке
2. Подтвердите удаление
3. Элемент будет удалён вместе с изображением (если было)

## API Endpoints

### Получение всех assets
```
GET /api/assets
Query parameters:
  - type: 'icon' | 'placeholder' (опционально)
  - category: string (опционально)

Response:
{
  "success": true,
  "data": [
    {
      "_id": "...",
      "name": "truck",
      "slug": "truck",
      "type": "icon",
      "url": "",
      "description": "Иконка доставки",
      "category": "delivery",
      "createdAt": "2026-05-13T...",
      "updatedAt": "2026-05-13T..."
    }
  ]
}
```

### Получение конкретного asset
```
GET /api/assets/:id
```

### Создание asset
```
POST /api/assets
Body:
{
  "name": "truck",
  "type": "icon",
  "description": "Иконка доставки",
  "category": "delivery"
}
```

### Обновление asset
```
PUT /api/assets/:id
Body:
{
  "name": "...",
  "type": "icon",
  "description": "...",
  ...
}
```

### Загрузка изображения
```
POST /api/assets/:id/image
Content-Type: multipart/form-data
Body:
  file: <binary image data>
```

### Удаление asset
```
DELETE /api/assets/:id
```

## База данных

### Коллекция: assets
```javascript
{
  _id: ObjectId,
  name: String (required),
  slug: String (required, unique),
  type: 'icon' | 'placeholder' (required),
  url: String (default: ''),  // Пустой для иконок, путь к файлу для заглушек
  description: String (optional),
  category: String (optional, for grouping),
  width: Number (optional, in pixels),
  height: Number (optional, in pixels),
  createdAt: Date,
  updatedAt: Date
}
```

### Индексы
- `slug` — уникальный индекс
- `type` — фильтрация по типу
- `category` — группировка по категориям

## Примеры использования

### Получить все иконки для платежей
```bash
curl http://localhost:3001/api/assets?type=icon&category=payment
```

### Получить все фото-заглушки
```bash
curl http://localhost:3001/api/assets?type=placeholder
```

## Ограничения и особенности

1. **Имена slug должны быть уникальными** — система использует slug на основе имени
2. **Иконки не требуют загрузки файлов** — они отображаются через Font Awesome CSS
3. **Заглушки требуют загрузки изображения** — файл сохраняется в `apps/api/uploads`
4. **Изображения хранятся в `apps/api/uploads/`** — папка создаётся автоматически
5. **Требуется аутентификация** — все операции требуют прав администратора
6. **Поддерживаемые форматы** — любые изображения (JPEG, PNG, GIF, WebP и т.д.)
7. **Максимальный размер файла** — 20 МБ

## Решение проблем

### Иконки не отображаются в админке
- Убедитесь, что `@fortawesome/fontawesome-free` установлен в `apps/admin`
- Проверьте импорт `@fortawesome/fontawesome-free/css/all.min.css` в `apps/admin/src/main.ts`

### Изображение не загружается
- Проверьте, что папка `apps/api/uploads` существует
- Убедитесь, что размер файла не превышает 20 МБ
- Проверьте формат файла (должен быть изображением)

### 401 Unauthorized
- Убедитесь, что вы авторизованы как администратор
- Проверьте, что токен JWT валиден

### Asset не появляется в списке
- Проверьте консоль браузера на ошибки
- Проверьте API логи сервера
- Убедитесь, что MongoDB подключена
- Запустите seed: `npx tsx apps/api/src/seed.ts`

### Битое изображение
- В админке битые изображения автоматически заменяются SVG-заглушкой с названием файла
