# Конфигурация портов KREPEJ

## Фиксированные порты (НЕ МЕНЯТЬ!)

| Приложение | Порт | URL | Файл конфигурации |
|------------|------|-----|-------------------|
| **API Server** | 3001 | http://localhost:3001 | `apps/api/.env` |
| **Admin Panel** | 3002 | http://localhost:3002 | `apps/admin/vite.config.ts` |
| **Web Frontend** | 3000 | http://localhost:3000 | `apps/web/vite.config.ts` |

## Важно!

- Все порты зафиксированы с помощью `strictPort: true` в Vite
- Если порт занят, приложение **НЕ запустится** - освободите порт перед запуском
- API прокси настроен на `http://localhost:3001` в обоих фронтендах
- Для Node.js v25+ используется `npx tsx` вместо `tsx watch`

## Быстрый старт

```bash
# Запустить все приложения
pnpm dev

# Или по отдельности:
pnpm dev:api    # API на порту 3001
pnpm dev:admin  # Админка на порту 3002
pnpm dev:web    # Сайт на порту 3000
```

## URLs для доступа

- **Сайт**: http://localhost:3000
- **Админка**: http://localhost:3002
- **API**: http://localhost:3001/api/health (проверка работоспособности)

## Если порт занят

### Windows:
```bash
# Найти процесс на порту
netstat -ano | findstr :3000

# Убить процесс (PID заменить на номер из вывода)
taskkill /PID <PID> /F
```

## Troubleshooting

1. **Не могу зайти в админку** - проверьте, что API запущен на порту 3001
2. **Не работает регистрация** - проверьте MongoDB и API лог
3. **Порт занят** - убейте процесс или измените порт в `.env` файле
